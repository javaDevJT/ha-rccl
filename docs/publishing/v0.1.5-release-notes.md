# Royal Caribbean `v0.1.5`

## Changes

- Keeps Royal Caribbean account reloads alive when RCCL intermittently returns
  a successful auth response without a login token.
- Continues to surface real credential failures, including HTTP 401/403, as
  Home Assistant authentication failures.
- Shrinks Club Royale offer-code sensor attributes so Home Assistant Recorder
  can store them without exceeding the 16,384 byte attribute limit.
- Replaces raw offer sailing arrays with counts and caps long display metadata
  lists on offer sensors.

## Notes

- Club Royale sailing data remains available through the sailing entities and
  custom card.
- Offer sensors still expose expiration dates and summary metadata, but no
  longer publish every raw sailing id or sail date as state attributes.
